# LUA协议结构注意事项

* 协议id,不能小于20000
* 协议类型,不要使用过长名字,尽量使用缩写
* 协议类型名,名称规则:client->server,以_tos结尾;server->client,以_toc结尾
* 文件名和消息名要一致
* 新增结构时,需要在 ** Conf/LuaProtoCfg.lua ** 中添加配置
* 数据结构暂不支持 int64/uint64类型,如果必须使用64位可使用字符串代替


# 哪些功能模块使用lua协议

* 和使用C#编写的功能模块耦合性低的
* 需要更新迭代玩法非内容的模块
* 生命周期短暂,随时上随时下的功能模块
* 以上仅作参考意见
